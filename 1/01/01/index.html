library(tidyverse)
library(janitor)

Janitor es una biblioteca que ayuda a limpiar nombres de columnas que vienen formateados de forma no ptima para el anlisis de datos en R. Generalmente sirve para aquellos Excel que nos pasan con nombres de columna con tildes, puntos como separadores, duplicados, entre otros problemillas.

Veamos un ejemplo (real) de un dataset, truncado por temas de confidencialidad, en el que una biblioteca como janitor nos vino al pelo.

```{r levanto la base, message=FALSE, warning=FALSE}

datos_originales <- read.csv(https://elradar.github.io/data/data_janitor.csv)

```

Al husmear la estructura del dataset, vemos que las columnas Gnero y Telfono tienen tilde, lo cual es muy correcto a nivel idiomtico pero muy desaconsejado a nivel pragmtico ya que tener caracteres especiales en el nombre de columnas puede afectar muchas operaciones que se encuentran penadas y adaptadas al ingls, o el esperanto de la era digital. Por otro lado, podemos ver que en muchas de las columnas han decidido usar el punto como separador. Esto tiene ms que ver con convencin (o hedonismo esttico) que por utilidad: pero en general se prefiere utilizar el guin bajo o underscore como separador. Por ltimo, que todo est en minscula no nos vendra del todo mal tampoco en caso de que querramos ahorrar tiempo si tipeamos (el viaje de ida y vuelta al Caps Lock).

```{r nombres_columnas, warning=FALSE}

datos_limpios <- datos_originales %>% clean_names()

colnames(datos_limpios)

```

Lo primero que salta a la vista es que janitor y tidyverse se llevan increblemente bien. Lo segundo, es que con una simple funcin todas las variables perdieron los caracteres especiales (en nuestro caso las tildes), los puntos fueron reemplazador por _" como separadores y todo est en bella letra minscula. Vamos a ver si hay filas enteramente vacas (todo es posible cuando viene de Excel), que tengan siempre el mismo valor (categora constante) y si hay alguna lnea duplicada.

```{r remuevo_filas, message=FALSE, warning=FALSE}

datos_limpios <- datos_originales %>% clean_names() %>% remove_empty(rows) %>% remove_constant() %>% convert_to_NA(factor)

duplicados <- datos_limpios %>% get_dupes()

dim(datos_limpios) dim(duplicados)

```

No tenemos filas vacas ni duplicados (tenemos la misma cantidad de observaciones) aunque s encontr una fila con todos valores constantes y la removi (fecha_ult_ingreso).

```{r recodear, message=FALSE, warning=FALSE}

datos_limpios <- datos_limpios %>% mutate(genero = recode(genero, A = hombre, B = mujer), apoyo_espiritual = recode(apoyo_espiritual, A = no quiso, B = tuvo apoyo))

```

Una gran ventaja de janitor es que nos permite adems es hacer tablas relativamente lindas de manera sencilla y pipe compatible. Vamos a cruzar la variable gnero con apoyo espiritual.

```{r tablas, message=FALSE, warning=FALSE}

datos_limpios %>% tabyl(genero, apoyo_espiritual)

```

Genial, tenemos la cantidad de casos en cada celda. Pero, no nos interesara mucho ms poder ver estos mismos datos como porcentajes?

```{r tablas_relativas, message=FALSE, warning=FALSE}

datos_limpios %>% tabyl(genero, apoyo_espiritual) %>% adorn_totals(row) %>% ##muestra el total por categoria de columna adorn_percentages(row) %>% ##nos muestra las celdas en porcentajes adorn_pct_formatting() %>% ##formatea el porcentaje de manera legible adorn_ns() %>% ##le suma a cada celda el nmero de casos absolutos entre parntesis adorn_title() ##le suma el ttulo del cruce en la esquina superior izquierda

```

Y as tenemos una tabla bastante linda, con bastante margen de formateo y sin salir del universo tidyverse (yey). Por supuesto que no sacaremos ninguna conclusin sobre estos datos ya que forman parte de una muestra no aleatoria ni representativa de un conjunto de informacin no disponible (an) para su publicacin.
